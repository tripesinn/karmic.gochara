# Migration Astro — Karmic Gochara

## Objectif
Remplacer le frontend Flask/Jinja (templates/index.html + static/app.js) par Astro. Backend REST reste sur Cloud Run — communication uniquement via fetch().

## Architecture cible
```
karmic-gochara/
├── astro/                    # Nouveau projet Astro
│   ├── src/
│   │   ├── layouts/
│   │   │   └── BaseLayout.astro   # Slot, meta, PWA, thème dark purple/gold
│   │   ├── pages/
│   │   │   └── index.astro        # Landing + login
│   │   ├── components/
│   │   │   ├── DailyReading.astro  # ◆ LIRE LE SIGNAL → fetch POST /calculate
│   │   │   └── ProUpgrade.astro    # Beta PRO → fetch POST /stripe/checkout
│   │   └── lib/
│   │       └── api.ts             # Client fetch() vers gochara-api
│   ├── public/
│   │   └── static/                 # manifest.json, icons PWA
│   └── astro.config.mjs
└── app.py                    # Backend inchangé — sert templates/ via Flask
```

## Workflow Gemma local

### Phase 1 — Squelettes (Gemma 4 E4B)
1. `src/layouts/BaseLayout.astro` — layout avec slot, meta viewport, PWA manifest, apple-touch-icon, thème dark purple (#0d0221 bg, #c9a84c gold), police mono titres
2. `src/pages/index.astro` — landing simple : titre + sous-titre + bouton login
3. `src/components/DailyReading.astro` — après login, bouton "◆ LIRE LE SIGNAL" → fetch POST /calculate, spinner, affiche résultat JSON dans une card. `client:load` pour l'interactivité
4. `src/components/ProUpgrade.astro` — card "GOCHARA PRO — Offert Beta", bouton "Activer PRO" → fetch POST /stripe/checkout {product_type:"subscription"}. Si {ok:true, beta:true} → reload
5. `src/lib/api.ts` — wrapper fetch() pour les routes API clés

### Phase 2 — Logique métier (deepseek/Claude)
- Gates d'accès (beta, subscription)
- Sessions utilisateur
- Routing API avancé
- Intégration Capacitor (PWA)

### Phase 3 — Tests
- Test sur navigateur + mobile
- Déploiement Cloud Run (static build)

## Contraintes
- Tailwind CSS uniquement
- Couleurs cosmiques : bg #0d0221, accent #c9a84c (gold)
- Pas de markdown — .astro avec `<script>` inline minimal
- Chaque fichier = un bloc de code unique prêt à copier
- Thème PWA déjà présent dans le dossier /www

## API existante (inchangée)
- `POST /api/auth` — login
- `POST /api/save_profile` — sauvegarde profil
- `POST /calculate` — synthèse astrologique
- `GET /api/profile` — session utilisateur
- `POST /stripe/checkout` — activation PRO
